# DARK Factory – VNB Assistant 기능 정의서

> 마지막 업데이트: 2026-06-23  
> 대상 파일: `frontend/src/components/vnb-assistant.tsx`, `center-workspace.tsx`, `data-sidebar.tsx`

---

## 1. 시스템 개요

| 항목 | 내용 |
|------|------|
| 시스템명 | DARK Factory – VNB Assistant |
| 목적 | 신한라이프 보험 상품의 VNB(Value of New Business) 예측 모델 학습·시뮬레이션·집계 플랫폼 |
| 주요 사용자 | 상품개발팀 |
| 기술 스택 | React + TypeScript + Vite · Tailwind CSS · shadcn/ui · Recharts |
| 배포 환경 | 로컬 Vite 개발서버 / 프론트엔드 단독 (백엔드 미연결 시 로컬 폴백 로직 내장) |

---

## 2. 네비게이션 구조

```
사이드바 (data-sidebar.tsx)
└── VNB Assistant
    ├── 학습용 DB          (id: training-db)
    ├── 예측용 DB          (id: prediction-db)
    ├── 예측결과 저장 DB   (id: result-db)
    ├── 예측모델 개발      (id: model-dev)
    ├── 시뮬레이션 및 집계 (id: simulation)
    ├── 대시보드           (id: vnb-dashboard)
    └── 출력              (id: output)
└── 기타 도구
    ├── 상품 설계 도구
    ├── VNB 시뮬레이터
    └── 상품 목록
```

## 3. 탭별 기능 정의

### TAB 1: 학습용 DB (`TrainingDbTab`)

**화면 ID**: `training-db`

| 항목 | 내용 |
|------|------|
| 주요 목적 | VNB 예측 모델 학습용 데이터 관리 및 생성 |

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 설명변수 DB 구축 및 관리 | 6개 카테고리(VNB결산정보·계약가입속성·상품정보·환산정보·최적가정·최소보험료) 변수 현황 테이블. 카테고리별 변수 수, 주요 필드, 연계 방식(실시간/정기배치), 동기화 상태 표시 |
| 2 | 내부 시스템 연계 | 계관·보험료산출·VNB결산·상품관리·분석계DW 5개 시스템 연계 방식 및 제공 주기 관리 테이블 |
| 3 | 학습 데이터 생성 | "학습데이터 대량 생성" 버튼 → Progress bar 시뮬레이션(100% 도달 시 완료 메시지). 월별 생성량 BarChart(Recharts) 표시 |
| 4 | 보종별 학습 데이터 현황 | 5개 보종별 학습 건수, 데이터 기간, 충분성(충분/보강필요) 표시 |
| 5 | 데이터 품질 지표 | 카테고리별 결측률·이상치 비율. 기준치(결측률 2%, 이상치 1%) 초과 시 amber 색상 경고 |
| 6 | 학습/검증 데이터 분리 현황 | Train 80% / Val 10% / Test 10% 비율 바차트로 보종별 시각화 |

**상태**: `isGenerating` (bool), `genProgress` (0~100)  
**상단 KPI**: 설명변수 총계 408개 · 학습 데이터 410만건 · 연계 시스템 5개

---

### TAB 2: 예측용 DB (`PredictionTab`)

**화면 ID**: `prediction-db`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 보종별 MP DB 생성 | 보종별 연결 모델 ID, MP 조합 기준(연령×성별×기간×금액×채널), 가입연령/보험기간/가입금액 범위, MP 건수, 준비 상태 테이블 |
| 2 | 보종별 VNB 예측 수행 | 보종·연령·성별·채널·가입금액 입력 → 해당 보종 예측 모델 자동 적용 → VNB/신계약마진율/환산APE 결과 표시. 예측 이력 테이블 포함 |

**상태**: `product`, `age`, `gender`, `channel`, `amount`, `result`  
**핵심 로직**: 보종마다 모델 ID가 1:1 매핑 (BY_CAT_MP 상수). 정기보험은 모델 미개발로 비활성화  
**상단 KPI**: 보종별 MP 총계 5,999만 · 전처리 완료 4종 · 최근 예측 10분 전

---

### TAB 3: 예측결과 저장 DB (`ResultDbTab`)

**화면 ID**: `result-db`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 보종별 MP 예측 결과 저장 | 보종별 연결 모델 ID, DB 테이블명(TB_VNB_RESULT_*), 저장 MP 건수, 최근 산출일, 적재 상태 표시 |
| 2 | 보종별 VNB 집계 결과 | MP별 예측 VNB에 판매 믹스(최근 3개월 실적) 가중 적용 후 보종 단위 집계. VNB·신계약마진율·환산APE 표시. 채널/가입조건/전사 합산 세부 집계 트리거 현황 포함 |

**상단 KPI**: 저장 MP 599만건 · DB 용량 38GB · 집계 완료 4종 · 최근 적재 10분 전

### TAB 4: 예측모델 개발 (`ModelDevTab`)

**화면 ID**: `model-dev`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 보종별 예측 모델 구축 현황 | 4개 운영 모델(M-CANCER-001, M-WHOLE-002, M-PENSION-003, M-HEALTH-004) 알고리즘, 정확도(1-MAPE), R², 상태, 최종 학습일 테이블. 정확도 컬럼에 (1-MAPE)×100 공식 툴팁 표시 |
| 2 | 보종별 알고리즘 선정 | Linear Regression(Baseline) / Random Forest / XGBoost / LightGBM / CatBoost 5가지 알고리즘 후보 및 특성 설명 |
| 3 | 모델 재현성 확보 | 보종 선택 시 모델 ID, 알고리즘, 버전, 전처리 파이프라인, MLflow Run ID, 하이퍼파라미터 전체 표시 (Select로 보종 전환) |
| 4 | 보종별 영향도 분석 (SHAP) | 보종 Select → 해당 보종 SHAP 변수 영향도 수평 BarChart 렌더링. 보종마다 주요 변수 순위 상이 |
| 5 | 보종별 성능 모니터링 | 보종 Select → 해당 모델의 월별 RMSE·MAPE 추이 LineChart |
| 6 | 재학습 체계 구축 | 정기(매주 월요일 02:00) / 성능저하(MAPE>3%) / 상품개정 이벤트 / 데이터드리프트(PSI>0.2) 4가지 자동 재학습 트리거 상태 표시 |
| 7 | 보종 개정 대응 | 요율개정·상품개정·이율변경 이력과 재학습 완료 상태 테이블 |

**상태**: `shapModel`, `perfModel`, `reproModel` (각각 보종 선택용)  
**상단 KPI**: 운영 모델 4개 · 평균 정확도 97.6% · 모니터링 실시간 · 재학습 트리거 주간

### TAB 5: 대시보드 (`AggregationTab`)

**화면 ID**: `vnb-dashboard`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | VNB 결산 DB vs 예측 모델 트렌드 | 보종 Select(전체/암보험/종신/연금/건강) → 분기별 결산 DB vs 예측 모델 LineChart. 미래 분기는 예측 모델만(점선) 표시, 결산 null 처리 |
| 2 | 동일가입조건 결산 DB vs 예측 모델 집계 | 보종 Select → 해당 보종의 동일 MP(가입조건) 기준 결산 DB · 예측 모델 · 차이 · MAPE 비교 테이블 |
| 3 | 원하는 조건으로 VNB 예측·집계 | 보종·채널·연령·성별·가입금액·보험기간·납기 입력 → 보종별 예측 모델로 VNB 즉시 산출, 결산 DB 동일조건 집계값과 나란히 비교 (차이·MAPE·신계약마진율 함께 표시) |
| 4 | 판매속성별 VNB 히트맵 | 행(↓)·열(→) 축을 채널/보종/연령대/성별/납기/보험기간 중 자유 선택. 선택 조합에 따라 가중치 곱(DIM_WEIGHTS) 기반 VNB 값 동적 계산·Blue 계열 색상 매핑 |

**상태**: `trendCat`, `cmpCat`, `simProduct`, `simAge`, `simGender`, `simChannel`, `simPeriod`, `simPayPd`, `simAmount`, `simResult`, `rowDim`(기본: 채널), `colDim`(기본: 보종)

**공유 상수**:
- `HEATMAP_DIMS`: 6개 차원(채널·보종·연령대·성별·납기·보험기간)별 값 목록
- `DIM_WEIGHTS`: 각 차원 값별 VNB 기여 가중치
- `quarterlyTrend`: 보종별 분기 결산 vs 예측 시계열 데이터
- `conditionCompare`: 보종별 동일가입조건 결산·예측 비교 데이터

**상단 KPI**: VNB Assistant 집계 527억 · 결산 DB 집계 514억 · 평균 MAPE · 집계 완료 4종

---

### TAB 6: 출력 (`OutputTab`)

**화면 ID**: `output`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 출력 설정 | 연도(2024~2026) + 분기(Q1~Q4/연간) + 집계 단위(증번/보종/상품/MIX/채널) + 출력 형식(Excel/PDF) + 보종 필터(전체/개별 체크박스) + 포함 항목(예측집계·결산비교·차트·트렌드·SHAP) 선택. Excel 선택 시 생성될 시트 목록 미리 표시 |
| 2 | 결과 미리보기 | 선택된 집계 단위·보종 필터에 따라 예측 모델 vs 결산 DB 테이블 동적 변경. 합계행 자동 계산. 보종 미선택 시 안내 메시지 |
| 3 | 출력 이력 | 이전 생성 파일 목록(생성일시·파일명·기간·집계단위·형식·크기·상태). 재다운로드 버튼 |

**상태**: `unit`, `format`, `year`, `quarter`, `selectedProds[]`, `inclChart/Model/Compare/Shap/Trend`

**다운로드 구현**:
- Excel: HTML 테이블을 `application/vnd.ms-excel` MIME으로 `.xls` 저장 → Excel에서 직접 열림
- PDF: `window.open()` 새 탭에 보고서 HTML 렌더링 → `window.print()` 자동 호출 (PDF로 저장 가능)

**공유 상수**:
- `OUTPUT_PREVIEW`: 집계 단위별 미리보기 데이터(보종/채널/상품/MIX/증번)
- `EXCEL_SHEETS`: 집계 단위별 Excel 시트 구성 목록
- `OUTPUT_HISTORY`: 출력 이력 5건
