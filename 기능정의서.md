# VNB Assistant 기능 정의서

> 마지막 업데이트: 2026-06-25  
> 대상 파일: `frontend/src/components/tabs/`, `layout/`, `pages/Home.tsx`

---

## 1. 시스템 개요

| 항목 | 내용 |
|------|------|
| 시스템명 | VNB Assistant |
| 목적 | 신한라이프 보험 상품의 VNB(Value of New Business) 예측 모델 학습·시뮬레이션·집계 플랫폼 |
| 주요 사용자 | 상품개발팀 |
| 기술 스택 | React 19 · TypeScript 5.7 · Vite 6 · Tailwind CSS 3 · shadcn/ui · Recharts 3.9 · react-grid-layout 2.2 |
| 배포 환경 | 로컬 Vite 개발서버 / Docker / GitHub Pages (백엔드 미연결 시 로컬 폴백 로직 내장) |

---

## 2. 레이아웃 구조

```
Home.tsx
├── 헤더 (신한라이프 로고 · VNB Assistant · 상품개발팀)
├── 좌측: DataSidebar (접기/펼치기 토글, 상단 "메뉴" 레이블)
├── 중앙: CenterWorkspace (탭 바 + 콘텐츠)
└── 우측: RightPanel (AI 어시스턴트 — 슬라이드 드로어)
     └── 플로팅 버튼 (우하단 고정, 챗봇 아이콘)
```

---

## 3. 네비게이션 구조

```
사이드바 (data-sidebar.tsx)
├── DB 관리 (기본 접힘, 하위 항목 점(•) 아이콘)
│   ├── 학습용 DB          (id: training-db)
│   ├── 예측용 DB          (id: prediction-db)
│   └── 예측결과 저장 DB   (id: result-db)
├── VNB예측모델 개발       (id: model-dev)
├── 시뮬레이션 및 집계     (id: simulation)
├── 대시보드               (id: vnb-dashboard)
├── 출력                   (id: output)
└── 기타 도구 (기본 접힘)
    ├── 상품 설계 도구     (id: product-builder)
    ├── VNB 시뮬레이터     (id: vnb-simulator)
    └── 상품 목록          (id: product-analysis-list)
```

## 4. 탭별 기능 정의

### TAB 1: 학습용 DB (`TrainingDbTab`)

**화면 ID**: `training-db`  
**파일**: `frontend/src/components/tabs/training-db-tab.tsx`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 설명변수 DB 구축 및 관리 | 6개 카테고리(VNB결산정보·계약가입속성·상품정보·환산정보·최적가정·최소보험료) 변수 현황 테이블 |
| 2 | 내부 시스템 연계 | 계관·보험료산출·VNB결산·상품관리·분석계DW 5개 시스템 연계 방식 및 제공 주기 관리 |
| 3 | 학습 데이터 생성 | "학습데이터 대량 생성" 버튼 → Progress bar 시뮬레이션. 월별 생성량 BarChart |
| 4 | 보종별 학습 데이터 현황 | 5개 보종별 학습 건수, 데이터 기간, 충분성 표시 |
| 5 | 데이터 품질 지표 | 카테고리별 결측률·이상치 비율. 기준치 초과 시 amber 경고 |
| 6 | 학습/검증 데이터 분리 현황 | Train 80% / Val 10% / Test 10% 비율 바차트 |

**상단 KPI**: 설명변수 408개 · 학습 데이터 410만건 · 연계 시스템 5개

### TAB 2: 예측용 DB (`PredictionTab`)

**화면 ID**: `prediction-db`  
**파일**: `frontend/src/components/tabs/prediction-tab.tsx`

| # | 섹션명 | 기능 |

**상단 KPI**: 보종별 MP 총계 5,999만 · 전처리 완료 4종 · 최근 예측 10분 전

### TAB 4: VNB예측모델 개발 (`ModelDevTab`)

**화면 ID**: `model-dev`  
**파일**: `frontend/src/components/tabs/model-dev-tab.tsx`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 보종별 예측 모델 구축 현황 | 4개 운영 모델 알고리즘, 정확도(1-MAPE), R², 상태, 최종 학습일 테이블 |
| 2 | 보종별 알고리즘 선정 | Linear/Random Forest/XGBoost/LightGBM/CatBoost 5가지 후보 및 특성 설명 |
| 3 | 모델 재현성 확보 | 보종 Select → 모델 ID·알고리즘·버전·MLflow Run ID·하이퍼파라미터 표시 |
| 4 | 보종별 영향도 분석 (SHAP) | 보종 Select → SHAP 변수 영향도 수평 BarChart |
| 5 | 보종별 성능 모니터링 | 보종 Select → 월별 RMSE·MAPE 추이 LineChart |
| 6 | 재학습 체계 구축 | 정기(매주 월)/성능저하(MAPE>3%)/상품개정/데이터드리프트(PSI>0.2) 4가지 트리거 상태 |
| 7 | 보종 개정 대응 | 요율개정·상품개정·이율변경 이력과 재학습 완료 상태 테이블 |

**상단 KPI**: 운영 모델 4개 · 평균 정확도 97.6% · 재학습 트리거 주간

### TAB 5: 시뮬레이션 및 집계 (`SimulationTab`)

**화면 ID**: `simulation`  
**파일**: `frontend/src/components/tabs/simulation-tab.tsx`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 집계 조회 | 집계 단위(증번/보종/상품/MIX/채널) Select → 예측 VNB vs 결산 VNB BarChart + 테이블 |
| 2 | 가입조건 VNB 시뮬레이션 | 보종·채널·연령·성별·가입금액·보험기간·납기 입력 → VNB 즉시 산출 |
| 3 | 변경 영향 분석 | 슬라이더로 판매MIX 9개 변수 조절 → 기준 VNB 대비 변화량 실시간 표시 |

**핵심 로직**:
- 집계 조회: `AGG_UNIT_DATA` 상수 → 단위별 예측·결산 정적 매핑
- 변경 영향: `CHG_BY_CAT[보종].coeff` × 슬라이더 변화량 → 총 델타 합산

### TAB 6: 대시보드 (`AggregationTab`)

**화면 ID**: `vnb-dashboard`  
**파일**: `frontend/src/components/tabs/aggregation-tab.tsx`

#### 글로벌 필터 바

대시보드 상단의 콤보박스 필터. 모든 위젯에 공통 적용.

| 필터 | 옵션 |
|------|------|
| 기준분기 | 2024Q1 ~ 2025Q4 |
| 보종 | 전체/암보험/종신/연금/건강 |
| 채널 | 전체/GA/방카/TM/대면 |

#### 위젯 그리드 (드래그앤드롭)

`react-grid-layout(legacy) + WidthProvider` 기반 자유 배치 그리드.

| 기능 | 설명 |
|------|------|
| 드래그 | 헤더(≡) 영역을 잡고 원하는 위치로 이동 |
| 리사이즈 | 우하단 핸들로 크기 조절 |
| 위젯 삭제 | 헤더 X 버튼 |
| 위젯 초기화 | "초기화" 버튼으로 기본 위젯 3종 복원 |

**기본 위젯 3종**:
- KPI 카드 (예측 VNB / 결산 VNB / 평균 MAPE)
- VNB 트렌드 LineChart (분기별 예측 vs 결산)
- 보종별 BarChart (예측 VNB)

#### 차트 빌더 (사용자 정의)

"차트 추가" 버튼으로 열리는 2단계 모달.

| 단계 | 내용 |
|------|------|
| Step 1 | 차트 유형 선택: 막대/선형/면적/원형/히트맵 |
| Step 2 | 축 구성: X축 차원(채널/보종/연령대/성별/납기/분기), Y축 지표(예측VNB/결산VNB/MAPE), 그룹 차원(선택) |

선택 완료 시 자동 제목 생성 후 그리드에 추가.

#### 데이터 모델

```
DimKey = 채널 | 보종 | 연령대 | 성별 | 납기 | 분기
MetricKey = forecast | settlement | mape

calcVnb(dims)     → 다차원 가중치 곱 기반 VNB 계산
aggregate1D(xDim) → 1차원 집계
aggregate2D(xDim, groupDim) → 2차원 그룹 집계
heatmapData(rowDim, colDim) → 히트맵용 2D 배열
```

### TAB 7: 출력 (`OutputTab`)

**화면 ID**: `output`  
**파일**: `frontend/src/components/tabs/output-tab.tsx`

| # | 섹션명 | 기능 |
|---|--------|------|
| 1 | 출력 설정 | 연도·분기·집계단위·출력형식(Excel/PDF)·보종필터·포함항목 선택 |
| 2 | 결과 미리보기 | 선택 조건에 따라 예측 vs 결산 테이블 동적 변경. 합계행 자동 계산 |
| 3 | 출력 이력 | 이전 생성 파일 목록. 재다운로드 버튼 |

**다운로드 구현**:
- Excel: HTML 테이블 → `application/vnd.ms-excel` MIME → `.xls` 저장
- PDF: 새 탭에 보고서 HTML 렌더링 → `window.print()` 자동 호출
